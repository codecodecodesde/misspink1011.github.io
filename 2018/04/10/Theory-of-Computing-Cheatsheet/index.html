4.1 Decidablility

UTM(Universal TM):
on input < M, w > where M is A TM and w is a given input. Simulate M on w. If M ver accepts or rejects, so do we.
L(U) = { < M, w >: M is a TM, w is an input, M accepts w }

4.2 Undecidability

There is a specific problem that is algorithmicaclly unsolvable.

undicidable language

The problem of determining whether a Turing machine accepts a given input string.
ATM = {< M, w> | M is a TM and M accepts w}
how to understand ATM: acceptance problem for TMs of testing whether a paticular TM accepts a given string can be expresse as a language. This language contains the encoding of all TMs together with strings that the DFAs accept.

Theorem: ATM is undicidable.
Proof: contradiction/diagonalization

  • suppose that H is a decider for ATM, where
    H accepts < M, w > exactly when M accepts w.
  • construct a new TM D. Run H on input < M, < M > >. output of D is opposite of what H outputs. That is, if H accepts, reject; and if H rejects, accept.
  • run D with its own description < D > as input. In that case, we get
    if D doesn’t accept < D >, D accept < D > ; and if D accepts < D >, D rejects < D >. contradiction!
  • neither TM D nore TM H can exist.
unrecognizable language

Theorem: A is decidable <==> A is recognizable and co-recognizable.(A’complement are recognizable)

5.1 Undecidable problems from language Theory

Reducibility plays an important role in classifying problems by decidability.
if A is reducible to B and B is decidable, A also is decidable; if A is undecidable and reducible to B, B is undicidable. The last version is key to proving that various problems are undecidable.

Halting Problem:
HALTTM = {< M, w > | M is a TM and M halts on input w}

ETM = {< M > | M is a TM and L(M) is empty set}
